Goal: Ensure all subscription plans correctly initiate Ozow payments and complete the full payment → subscription activation flow

Audit and fix the Ozow subscription payment flow so that every subscription option (Monthly / Annual / Lifetime):

Redirects the user to Ozow checkout

Supports Visa, Mastercard, Instant EFT, Bank Apps, Google Pay (if enabled by Ozow)

Activates the subscription only after confirmed payment

Is secure, server-validated, and production-ready

1️⃣ Frontend (Paywall / Subscribe button)

Ensure Subscribe Now:

Calls a backend endpoint (NOT Ozow directly)

userId

planId (monthly | annual | lifetime)

Prevent price manipulation from frontend

2️⃣ Backend (Ozow integration)

Verify /api/ozow/create-payment exists and:

Generates server-side transaction reference

Uses server-side pricing

Builds Ozow hash correctly using:

OZOW_SITE_CODE

OZOW_PRIVATE_KEY

OZOW_API_KEY

Redirect user to Ozow payment URL

3️⃣ Ozow Callbacks

Ensure all callbacks exist and work:

/api/ozow/callback (payment notification)

/api/ozow/success

/api/ozow/cancel

/api/ozow/error

Verify Ozow hash/signature

Reject invalid or unknown transactions

Update payment status only after verification

payments

reference

status

subscriptions

user_id
